1 // author Rob Bateman [mailto:robthebloke@hotmail.com] [http://robthebloke.org]
2 // date 11-4-05
3 // brief A C++ loader for the alias wavefront obj file format. This loader is
4 // intended to be as complete as possible (ie, it handles everything that
5 // Maya can spit at it, as well as all the parametric curve and surface
6 // stuff not supported by Maya's objexport plugin).
7 //
8 // If you want to add obj import/export capabilities to your applications,
9 // the best method is probably to derive a class from Obj::File and then
10 // use the data in the arrays provided (or fill the data arrays with the
11 // correct info, then export).
12 //
13 // In addition to simply accessing the data arrays, you can also overload
14 // the functions, PreSave,PostSave,PreLoad,PostLoad and HandleUnknown to
15 // extend the capabilities of the file format (if you really think that
16 // is wise!).
17 //
18 // Generally speaking, the obj file format is actually very very complex.
19 // As a result, not all of the Curve and Surface types are fully supported
20 // in the openGL display parts of this code. Bezier, Bspline & NURBS curves
21 // and surface should be fine. Cardinal, taylor and basis matrix types
22 // will be displayed as CV's / HULLS only. Trim curves and holes don't get
23 // displayed either. I'll leave that as an excersice for the reader.
24 //
25 // The material lib files (*.mtl) are also supported and parsed, this
26 // includes a user extensible interface to load the image files as textures.
27 // See the texture manager example for more info.
28 //
29 // One nice aspect of this code is the ability to calculate normals for
30 // the polygonal surfaces, and the ability to convert the objfile into
31 // surfaces and /or vertex arrays. Hopefully it may be useful to someone.
32 //
33 // If you wish to use this code within a game, I **STRONGLY** RECOMMEND
34 // converting the data to Vertex Arrays and saving as your own custom
35 // file format. The code to handle your own objects will be substantially
36 // more efficient than this code which is designed for completeness, not
37 // speed! See the obj->game converter for a quick example of this.
38 //
39 //

110  SIMUL_OPENGL_EXPORT_STRUCT VertexParam
111  {
113  float u;
115  float v;
117  float w;
119  VertexParam(std::istream& ifs)
120  {
121  char buffer[128];
122  ifs.getline(buffer,128);
123  char*ptr = buffer;
124  while(*ptr==' '||*ptr=='\t')
125  ++ptr;
126  char*ptr2 = ptr;
127  int n=1;
128  while (*ptr2!='\0')
129  {
130  if (n==' '||n=='\t') {
131  ++n;
132  }
133  ++ptr2;
134  }
135  switch(n)
136  {
137  case 2:
138  sscanf(buffer,"%f%f",&u,&u);
139  w=0;
140  break;
141  case 3:
142  sscanf(buffer,"%f%f%f",&u,&v,&w);
143  break;
144  default:
145  break;
146  }
147  }
149  VertexParam()
150  : u(0),v(0),w(0)
151  {
152  }
154  VertexParam(const VertexParam& v_)
155  : u(v_.u),v(v_.v),w(v_.w)
156  {
157  }
159  friend std::ostream& operator << (std::ostream& ofs,const VertexParam& v_)
160  {
161  return ofs << "vp " << v_.u << " " << v_.v << " " << v_.w << std::endl;
162  }
163  bool operator==(const VertexParam& v_) const
164  {
165  return FCOMPARE(u, v_.u)&&
166  FCOMPARE(v, v_.v)&&
167  FCOMPARE(w, v_.w);
168  }
