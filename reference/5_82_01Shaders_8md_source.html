1 5.2 Shaders {#man_5_classes_ooo2_shaders}
2 =======
3 
4 trueSKY uses *effects* for rendering. An effect is a collection of shaders - vertex shaders, pixel shaders and so on, along with information on
5 how to combine them, how to set render state when they are used, and what inputs they take.
6 
7 Platform-specific derived classes of \link simul::crossplatform::Effect Effect\endlink are created with \link simul::crossplatform::RenderPlatform::CreateEffect RenderPlatform::CreateEffect\endlink,
8 usually with an extensionless filename passed to the function.
9 
10 The RenderPlatform first looks for a platform-specific shader source file by appending its platform shader file extension - .fx effect files for DirectX,
11 .glfx files for GLSL etc. If this is not found, the cross-platform .sfx extension is appended.
12 
13 The \#include directive has been enabled - DirectX handles this by design; while
14 Simul has implemented \#include for GLSL. Platform-specific include files have the suffix .hlsl, .glsl, etc., while cross-platform includes have extension .sl.
15 
16 As far as possible, the functional aspects of shading are defined
17 in cross-platform shader include files, which have been given the .sl suffix,
18 and can be found in the Simul/Platform/CrossPlatform/SL directory.
19 
20 In order for the different shader languages to use these common files, each
21 platform has a special file that is included before any .sl files, and which
22 contains definitions needed for the .sl files, including macros and functions - see CppHlsl.hlsl, etc.
23 
24 Constant Buffers
25 ----------------
26 Older versions of GLSL, and DirectX 9, pass constants to shaders individually.
27 
28  (in HLSL)
29  float brightness;
30 
31  (in C++)
32  D3DXHANDLE h=effect->GetParameterByName(NULL,"brightness");
33  effect->SetFloat(h,1.0);
34 
35 OpenGL 3.0 and DirectX 11 have the capability to use \em constant \em buffers instead,
36 so if we define in the shader:
37 
38  (in GLSL)
39  layout(std140) uniform ShaderConstants
40  {
41  uniform float brightness;
42  uniform float gamma;
43  uniform float bloom;
44  };
45 
46 etc., and in C++:
47 
48  struct ShaderConstants
49  {
50  float brightness;
51  uniform float gamma;
52  uniform float bloom;
53  };
54 
55 it is then possible to set the values of the whole structure in C++, and pass it complete to the shader. For cross-platform constant buffers, we define:
56 
57  SIMUL_CONSTANT_BUFFER
58 
59  SIMUL_CONSTANT_BUFFER_END
60 
61 and these macros have different expansions depending on whether the header is being used in C++, or in a shader compilation.
