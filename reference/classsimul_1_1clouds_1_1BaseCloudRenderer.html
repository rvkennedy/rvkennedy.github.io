Detailed Description

Base class for real-time volumetric cloud renderers. Source code is provided for this class Derive a new renderer from this base class, or use SimulCloudRenderer or SimulGLCloudRenderer.

The usual way to do this is to interpolate between two ready-made cloud textures, while preparing a third for use.

Interpolating cloud textures based on key-frames

This is done by the simul::clouds::CloudKeyframer class, an instance of which is stored in the simul::clouds::BaseCloudRenderer. The cloud renderer uses iterators from the CloudKeyframer to fill in texture data.

The cloud volume is calculated in keyframes, and the closer together in real time those keyframes are, the more CPU or GPU time will be spent on cloud calculation. Aside from any changes to the cloud grid data, the cloud volume can move horizontally and vertically at little cost. Typically, the CloudKeyframer instance stored in the Environment object is used to control both of these aspects.

Once per frame, the cloud renderer's PreRenderUpdate function is called from BaseWeatherRenderer::PreRenderUpdate, and any needed updates to the textures are, performed.

The keyframer also controls cloud motion due to wind, using WindSpeed and WindHeadingDegrees. Use SetOverrideWind() to choose whether to take wind values from the keyframes, or set them as needed.

In the shaders, the cloud interpolation value from the keyframer is used to blend between textures 0 and 1, while texture 2 is being built - see below.

Rendering Method

The cloud data consists of a 3D block of cloud values, GridLength*GridWidth*GridHeight.

The Cloud Grid

The viewpoint can be anywhere - below, above, even inside this volume. If SetWrap was called, the clouds wrap - the volume can is tiled in the horizontal directions.

To draw the clouds, the space around the viewpoint is divided up by a latitude-longitude spherical shell, centered on the viewpoint, but always aligned with the world axes.

The integer elevation_grid gives the number of latitude sections, while azimuth_grid gives the number of longitude segments. There is one of these shells for each layer we render through the cloud geometry - rendering from back to front with transparency:

For efficiency, only the parts of each shell that intersect the cloud geometry need to be drawn.

And we only draw the parts of each shell that intersect the camera view frustum.

Textures and Shaders

The keyframer uses the CPU or GPU cloud generator (e.g. simul::clouds::BaseGpuCloudGenerator) to fill volumetric data in GPU memory.

The final volume textures by default use their four channels as follows:

x red secondary light
y green primary light
z blue density
w alpha ambient light

Depending on the platform, rendering is done with a single pass using a raycasting shader, or using generated vertex geometry. A cloud depth buffer is filled in if needed.

The final cloud colour is composed of the primary and secondary (lightResponse.w*density.y) components, both multiplied by the sunlight colour, added to the ambient colour:

float3 cloud_colour=(density.z*Beta+lightResponse.w*density.y)*sunlightColour+ambient.rgb;

In bright daylight, only primary and secondary will usually be important, but towards sunset and in twilight, the ambient colour will be stronger.

While the keyframer is used to control the cloud object's realtime properties, other, permanent properties may be initialized in code, such as, to treat the volume as being wrapped horizontally:

cloudProperties->SetWrap(true);

The cloud's grid sizes:

cloudProperties->SetGridLength(128);
cloudProperties->SetGridWidth(128);
cloudProperties->SetGridHeight(8);

The cloud dimensions can be initialized; a typical cloud volume might extend 50km horizontally, and 4000m high.

cloudProperties->SetCloudWidth(50000.f);
cloudProperties->SetCloudLength(50000.f);
cloudProperties->SetCloudHeight(4000.f);

If the Wrap property is true, the clouds stretch infinitely in all directions - then Cloud Length and Width define how far you have to look to see the clouds repeating. If these distances are near to or larger than the maximum viewing distance, no repetition will be apparent.

The clouds are defined be a 3D box - the height of this box determines the thickness of the cloud layer, while the width and length determine its extent horizontally. If the cloud volume wraps, it has no limit in the horizontal directions, then the length and width determine how far you would need to move before the cloud pattern repeats. To enable wrapping, use simul::clouds::CloudProperties::SetWrap.

The next setting to choose is how much 3D detail to have in the clouds. The cloud volume is a grid, and you can set the grid shape using SetGridLength, SetGridWidth and SetGridWidth. Again, it is usual to have equal width and length, while height will be smaller, a 128x128x16 grid is typical.
