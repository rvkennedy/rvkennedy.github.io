Textures are assumed to be 32 bits per texel. SetBits determines which bits will be filled. For a noise texture, x, y, and z are the 3 elements of the noise value to be returned, and w is unused. For a cloud texture, density is x, directional light brightness y, secondary light z, and ambient light w. Bytes_per_texel is the number of bytes per texel, e.g. 2 for a 16 bit texture, 4 for a 32 bit texture etc. bigendian specifies whether this platform stores numbers in reverse byte-order. The Xbox 360 and PS3 both do this, while the PC does not.
