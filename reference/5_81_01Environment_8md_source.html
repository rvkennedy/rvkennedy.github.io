1 # 5.1 Controlling the Environment {#man_5_classes_ooo1_environment} #
2 
3 ## Updating ##
4 The \link simul::clouds::Environment Environment\endlink object stores an instance of \link simul::sky::SkyKeyframer SkyKeyframer\endlink
5 and two instances of \link simul::clouds::CloudKeyframer CloudKeyframer\endlink, and performs the necessary initialization and updates.
6 
7 As described in \ref man_2_gettingstarted, you will declare and create an instance of Environment. If you have a sequence file (.sq or .seq)
8 saved from the \ref man_6_sequencer, you can load this into the Environment and its keyframers will be appropriately initialized with global values
9 and keyframes.
10 
11  simul::clouds::Environment *environment=NULL;
12  ...
13  environment=new simul::clouds::Environment();
14 
15 You can either load a preconfigured sky sequence (.sq file), control the environment direct from the trueSKY API, or both together.
16 To load a file (the .sq is a human-readable text file format), create a TextFileInput object.
17 
18  simul::crossplatform::TextFileInput ifs;
19  ifs.Load("test_file.sq");
20  environment->LoadFromText(ifs);
21 
22 You can use your own text loader class in place of TextFileInput.
23 
24 The properties and keyframes of each Keyframer can also be modified in real time.
25 The keyframes each have a \a time value, and the keyframer tracks a current time through these frames, interpolating properties as it does so. This time value is obtained
26 or set using the skyKeyframer:
27 
28  float time=environment->skyKeyframer->GetTime();
29 
30 You don't need to update the keyframers specifically, so long as you're using simul::clouds::Environment::Update() once per frame.
31 
32 For more information on the two types of keyframer, see:
